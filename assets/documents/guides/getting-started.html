By default Global Geocoding SDK on premise solution does not have data required to use API's. So, before you can start geocding you must install datasets.

Step 1: Running CLI

On windows run the cli.cmd and on any linux/unix system run the cli.sh

cli can also be run in batch mode by creating a file with all commands that needs to be executed.

commands.cli

extract --s "/spd/download/location" --d "/spd/extract/location"
configure --s "/spd/extract/location" --d "/ggs/configuration/location"

Running all the commands inside commands.cli in batch command mode

./cli.sh --cmdfile=commands.cli

Step 2: Extracting datasets 

extract --s "/spd/download/location" --d "/spd/extract/location"

Parameters

Parameter Description
--s Directory containing .spd files or .spd file. If directory is provided then all .spd files from the given directory will be extracted.
--d Destination directory where the data will be extracted.

Step 3: Configuring datasets 

configure --s "/spd/extract/location" --d "/ggs/configuration/location"

Parameters

Parameter Description
--s Directory or comma seperated list of directories where data has been extracted.
--d Directory where configuration file should be created. Any existing configuration will be overwritten.
